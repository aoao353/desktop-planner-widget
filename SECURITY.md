# 安全策略

## 支持的版本

| 版本 | 是否接收安全更新 |
|------|------------------|
| 当前 `main` 分支上的最新代码 | 是 |
| 已发布的最新稳定版本 | 是 |
| 更旧的次要版本 | 视情况评估 |

## 报告漏洞

若你发现 **task-widget** 中的安全漏洞，请：

1. **不要**通过公开 GitHub Issue 披露利用细节或 PoC。  
2. 优先通过 [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories/about-github-security-advisories)（若仓库已开启）或仓库维护者提供的私密渠道联系。  
3. 在报告中尽量包含：复现步骤、受影响版本/提交、潜在影响范围。

维护者会在合理时间内确认收到并协调修复与发布说明；在修复前请勿对外公开细节。

## 范围说明

以下情况通常不在安全漏洞范围内，但可通过普通 Issue 反馈：

- 依赖项的已知 CVE（请优先向对应上游报告并跟进升级）  
- 仅影响本地开发环境、且需本机权限才能触发的配置问题  

感谢你对项目安全的关注。
